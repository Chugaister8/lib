МОДУЛЬ 0 · РОЗДІЛ 1: Що таке програмування і як думати як розробник
§ 0.1.3 Що таке JavaScript і де він застосовується

─────────────────────────────────────────────────────────────────────


Кожна технологія має свій момент народження. Більшість із них приходить
у світ поступово: роками розробляються специфікації, проводяться
конференції, пишуться дисертації — і лише потім з'являється версія 1.0.
Але JavaScript народився інакше. Різко, майже випадково, за десять днів —
і одразу змінив усе. Не тому що був ідеальним. А тому що опинився в
потрібному місці в потрібний момент.

Сьогодні JavaScript — це мова, яку використовує 98% усіх вебсайтів у
світі. Понад тридцять мільйонів розробників щодня пишуть на ній код.
Вона однаково запускається у браузері, на сервері, у мобільному додатку,
у настільній програмі, на мікроконтролері і в хмарній функції. Немає
жодної іншої мови програмування, яка охоплює настільки різні середовища
виконання, залишаючись при цьому тією самою мовою.

Але як усе це починалося? Що насправді являє собою JavaScript як мова?
І де саме — у якому рядку коду, у якому файлі, на якому пристрої — він
живе та діє? Цей розділ відповідає на всі ці питання послідовно й
вичерпно, від першого рядка коду 1995 року до сучасної екосистеми,
яка обіймає весь стек сучасної розробки.


НАРОДЖЕННЯ JAVASCRIPT: ДЕСЯТЬ ДНІВ, ЩО ЗМІНИЛИ ВСЕ

Щоб зрозуміти JavaScript, потрібно зрозуміти контекст, у якому він
народився. 1994 рік. Інтернет вибухнув — але не так, як ми звикли
уявляти. Вебсторінки того часу були суто статичними: HTML-документи,
завантажені один раз, без жодного руху, без жодної реакції на дії
користувача. Клікнув кнопку — нічого не трапилося. Заповнив форму —
сторінка перезавантажилася цілком. Жодної інтерактивності, жодного
відчуття живого інтерфейсу.

Компанія Netscape, яка у той час контролювала ринок браузерів, бачила
цю проблему і розуміла, що її треба вирішити. Потрібна була мова, яка
запускалася б безпосередньо у браузері і дозволяла сторінці реагувати
на дії користувача без перезавантаження. Щось просте, доступне навіть
для тих, хто не є програмістом у класичному розумінні — веб-дизайнерам,
верстальникам, авторам контенту.

У 1995 році до Netscape найняли молодого інженера на ім'я Брендан Айк.
Завдання було конкретним: створити нову мову програмування для браузера.
Термін — жорсткий. Айк виконав завдання за десять днів. Десять. Це не
перебільшення і не легенда — це задокументований факт, яким сам Айк
ділився неодноразово. Перша версія мови називалася LiveScript і була
грубою, з очевидними компромісами, на які довелося піти через час.
Але вона працювала.

Тут у гру вступає маркетинг — і трохи іронії. Наприкінці 1995 року
Java від Sun Microsystems була надзвичайно популярною. Кожен чув про
Java. Кожна компанія хотіла Java. І Netscape вирішила скористатися
цим хвилею: LiveScript перейменували на JavaScript. Рішення суто
маркетингове — жодного технічного зв'язку між двома мовами не
існувало й не існує досі. JavaScript і Java — це дві принципово різні
мови з різною архітектурою, різними парадигмами, різним призначенням.
Але назва прижилася, і спільнота досі жартує: «Java та JavaScript
пов'язані так само, як автомобіль та килимок».

1997 рік приніс важливу подію: ECMA International — міжнародна
організація зі стандартизації — затвердила офіційний стандарт мови.
Офіційна назва стандарту — ECMAScript. Саме тому сьогодні ви часто
бачите позначення ES6, ES2015, ES2020 та подібні: це версії стандарту
ECMAScript, які вказують, якого рівня мовні можливості підтримуються.
JavaScript — це реалізація стандарту ECMAScript, і між цими двома
назвами є тонка, але важлива різниця: ECMAScript — це специфікація,
а JavaScript — це конкретна мова, яку використовує код у браузері.

Довгий час JavaScript залишався суто браузерною технологією. Так тривало
аж до 2009 року, коли американський розробник Раян Дал ухвалив рішення,
що перевернуло індустрію. Він взяв Google V8 — JavaScript-рушій, який
Chrome використовував для виконання коду у браузері — і виніс його
за межі браузера, побудувавши навколо нього нове середовище виконання.
Так народився Node.js. Відтепер JavaScript міг запускатися на сервері,
читати файли, керувати мережею і виконувати практично ті самі функції,
що й будь-яка «серверна» мова на кшталт PHP, Ruby або Python.

Нарешті, починаючи з 2015 року — з версії ES6, або ES2015 — мова
отримала потужне оновлення, яке остаточно перетворило її на зрілий
інструмент. З'явилися класи у сучасному синтаксисі, модулі для
організації коду, стрілкові функції, конструкція async/await для
роботи з асинхронними операціями, деструктуризація, шаблонні рядки,
оператор розпакування та багато іншого. Мова, яка народилася за десять
днів як інструмент для веб-дизайнерів, перетворилася на одну з
найпотужніших і найуніверсальніших мов у сучасному програмуванні.


ЩО ТАКЕ JAVASCRIPT: ОФІЦІЙНЕ ВИЗНАЧЕННЯ І КЛЮЧОВІ ВЛАСТИВОСТІ

Якщо відкрити MDN — Mozilla Developer Network, авторитетне джерело
документації для вебтехнологій — можна знайти таке визначення:
JavaScript — це легка, інтерпретована мова програмування з першокласними
функціями. Вона найбільш відома як мова сценаріїв для вебсторінок,
але також використовується в багатьох небраузерних середовищах,
наприклад Node.js.

Кожне слово цього визначення несе змістовне навантаження. Розберімо
їх окремо, бо за кожним стоїть принципово важлива властивість мови.

«Легка» — означає, що JavaScript не вимагає складної інфраструктури
для запуску. Браузер вже має рушій. Node.js встановлюється за одну
команду. Перший код можна запустити у консолі DevTools без жодних
налаштувань, IDE або компіляторів.

«Інтерпретована» — це характеристика способу виконання. На відміну
від мов, які компілюються заздалегідь у машинний код (як C++ або
Rust), JavaScript традиційно виконується рядок за рядком під час
роботи програми. У сучасних реалізаціях це значно спрощено завдяки
JIT-компіляції, про яку ми детально говорили у попередньому розділі,
але принципова модель залишається: код не потребує окремої фази
компіляції перед запуском.

«Першокласні функції» (first-class functions) — це технічна
характеристика, яка означає, що функції у JavaScript є повноцінними
значеннями. Їх можна зберегти у змінній, передати як аргумент до
іншої функції, повернути як результат із функції, зберегти у масиві
чи об'єкті. Це фундаментальна властивість, яка робить можливим
функціональне програмування у JavaScript і є ключовою для розуміння
колбеків, замикань та вищих функцій.

Крім цих трьох характеристик із офіційного визначення, у JavaScript
є ще кілька ключових властивостей, які визначають його природу.

Мультипарадигмовість — JavaScript не прив'язаний до жодної однієї
парадигми програмування. У ньому можна писати в об'єктно-орієнтованому
стилі з класами та успадкуванням, у функціональному стилі з чистими
функціями та незмінними даними, і в процедурному стилі — послідовно,
команда за командою. Ба більше, у реальному коді всі три парадигми
часто поєднуються в межах одного файлу, і це не вважається поганою
практикою. Така гнучкість — і перевага, і джерело труднощів для
початківців, яким треба знайти власний стиль.

Однопотоковість — JavaScript виконується в одному потоці (thread).
Це означає, що у будь-який момент часу виконується рівно одна операція.
Немає паралельних потоків, немає гонки за даними між потоками — немає
і класу проблем, з якими розробники на Java або C++ ламають голови.
Але постає питання: якщо мова однопотокова, як вона справляється з
мережевими запитами, таймерами, анімаціями? Відповідь — Event Loop,
детально описаний у попередньому розділі. Асинхронність у JavaScript
реалізована без справжнього паралелізму, через чергу подій — і це
є однією з найцікавіших архітектурних рис мови.

Динамічна типізація — тип змінної у JavaScript не оголошується явно
і визначається автоматично під час виконання, залежно від значення,
яке їй присвоєно. Можна написати let x = 5, і x стане числом. Потім
x = 'текст' — і x стане рядком. Ніяких оголошень, ніяких компіляторних
перевірок. З одного боку, це спрощує написання коду. З іншого — є
джерелом помилок, які важко відловити без спеціальних інструментів.
Саме для вирішення цієї проблеми компанія Microsoft розробила TypeScript —
надбудову над JavaScript зі статичною типізацією, про яку ми поговоримо
детально трохи згодом.


JAVASCRIPT СЕРЕД ІНШИХ МОВ ПРОГРАМУВАННЯ

Будь-яку технологію краще розуміти у порівнянні. JavaScript — не
єдина мова програмування у світі, і знати, чим він відрізняється
від інших, де перетинається і де розходиться — практично корисно
і концептуально важливо.

За даними Stack Overflow Developer Survey 2024 — найбільшого щорічного
дослідження серед розробників у світі — JavaScript посідає перше місце
за популярністю вже дванадцять років поспіль. Це само по собі
вражаючий факт, але ще важливіше зрозуміти, які саме мови перебувають
поруч і чим кожна з них відрізняється.

Python займає другу позицію рейтингу. Як і JavaScript, Python є
динамічно типізованою мультипарадигмовою мовою. Але сфери застосування
суттєво різняться: Python домінує у науці про дані, машинному навчанні,
автоматизації та бекенді, але майже відсутній у браузерній розробці —
тій частині, де JavaScript абсолютно незамінний. Є спроби запускати
Python у браузері через технологію WebAssembly, але вони залишаються
нішевими й не загрожують позиції JavaScript у фронтенді.

C++ займає третю сходинку. Ця мова зі статичною типізацією і
мультипарадигмовою природою є мовою вибору для ігрових рушіїв,
операційних систем, драйверів та систем з жорсткими вимогами до
продуктивності. C++ компілюється у нативний машинний код без жодного
проміжного шару, тому є значно швидшою за JavaScript у суто
обчислювальних завданнях. Але написати вебзастосунок на C++ — завдання
не просто складне, а практично позбавлене сенсу: для цього є JavaScript.

Java займає четверту позицію. Попри співзвучність назв — маркетингову
помилку, яка плутала людей із 1995 року — Java і JavaScript є
принципово різними мовами. Java статично типізована, компілюється
у байткод JVM (Java Virtual Machine) і є суворо об'єктно-орієнтованою.
Її застосовують у корпоративному бекенді та розробці для Android.
JavaScript, навпаки, динамічний, мультипарадигмовий і значно
гнучкіший у синтаксисі.

TypeScript займає п'яту сходинку — і це окремий, дуже цікавий випадок.
TypeScript — це не самостійна мова у повному розумінні, а надбудова
над JavaScript зі статичною типізацією. Будь-який валідний JavaScript-код
є водночас валідним TypeScript-кодом. TypeScript компілюється у звичайний
JavaScript і широко застосовується у великих проєктах, де суворий
контроль типів допомагає уникати помилок. Про TypeScript ми поговоримо
детально в окремому підрозділі цього розділу.

Що ця картина говорить про JavaScript загалом? Мова перебуває у першій
п'ятірці за популярністю — але, на відміну від більшості конкурентів,
вона охоплює весь стек розробки: і браузер, і сервер, і мобільний
додаток, і настільну програму. Жодна інша мова з цього списку не
претендує на таку широту.


JAVASCRIPT У БРАУЗЕРІ: ЩО ВІН ВМІЄ

Браузер — це рідне середовище JavaScript. Саме тут мова народилася,
саме тут вона отримала всі ті можливості, які зробили сучасний веб
таким, яким ми його знаємо. Розберімо конкретно, які завдання
JavaScript виконує у браузері.

Маніпуляції з DOM — це найфундаментальніша функція браузерного
JavaScript. DOM (Document Object Model, модель об'єктів документа) —
це програмний інтерфейс, через який JavaScript отримує доступ до
структури HTML-сторінки. Кожен елемент сторінки — заголовок, абзац,
кнопка, зображення, форма — є об'єктом у DOM-дереві. JavaScript може
знайти будь-який із цих об'єктів, прочитати його властивості, змінити
вміст, стилі або атрибути, додати нові елементи або видалити існуючі —
і все це відбувається в реальному часі, без перезавантаження сторінки.

Обробка подій — другий ключовий аспект браузерного JavaScript. Коли
користувач клацає кнопку, рухає мишею, натискає клавішу, прокручує
сторінку, відправляє форму або завантажує файл — браузер генерує
подію (event). JavaScript може «підписатися» на будь-яку подію і
виконати довільний код у відповідь. Саме ця механіка лежить в основі
будь-якого інтерактивного вебінтерфейсу.

Розглянемо конкретний авторський приклад, який демонструє обидві
ці можливості разом:

    const knyzhkaBtn = document.querySelector('#dobraKnyzhka');

    knyzhkaBtn.addEventListener('click', () => {
      const povidomlennya = document.createElement('p');
      povidomlennya.textContent = 'Дякую, що читаєш!';
      povidomlennya.style.color = '#2a9d8f';
      document.body.appendChild(povidomlennya);
    });

Цей код знаходить кнопку з ідентифікатором dobraKnyzhka, підписується
на подію кліку і щоразу при натисканні створює новий абзац із текстом
і додає його до сторінки. Жодного перезавантаження. Жодних запитів
до сервера. Лише JavaScript, DOM і подія.

Мережеві запити — третя суттєва можливість. Сучасний веб неможливо
уявити без динамічного завантаження даних: стрічка новин, яка
оновлюється без перезавантаження, пошукові підказки, що з'являються
в процесі введення, кошик інтернет-магазину, який синхронізується
з сервером. Усе це реалізується через fetch() — сучасний браузерний
API для HTTP-запитів — або через старіший XMLHttpRequest. JavaScript
надсилає запит до сервера у фоновому режимі, отримує відповідь і
оновлює сторінку. Ця техніка має окрему назву — AJAX (Asynchronous
JavaScript and XML), хоча сьогодні замість XML частіше використовують
JSON.

Зберігання даних безпосередньо у браузері — ще одна можливість
JavaScript. Є кілька механізмів: localStorage зберігає дані без
обмеження часу між сесіями (наприклад, налаштування теми або мова
інтерфейсу), sessionStorage зберігає дані лише протягом поточної
сесії браузера, IndexedDB — це повноцінна вбудована база даних
у браузері для зберігання великих обсягів структурованих даних,
а cookies — найстаріший механізм, що використовується переважно
для аутентифікації.

Анімації та графіка — JavaScript також керує усім, що рухається і
малюється у браузері. Через CSS-анімації, керовані JavaScript,
реалізують перехідні ефекти та анімовані інтерфейси. API Canvas
дозволяє малювати двовимірну графіку прямо у браузері за допомогою
JavaScript-коду. WebGL — більш потужний інтерфейс — дає змогу
рендерити тривимірну графіку з апаратним прискоренням. Саме на
WebGL побудовані браузерні ігри, інтерактивні візуалізації даних
і навіть деякі CAD-інструменти.


DOM — DOCUMENT OBJECT MODEL: ДЕРЕВО СТОРІНКИ

DOM заслуговує на окреме, докладне пояснення, бо це поняття є
центральним для розуміння того, як JavaScript взаємодіє із вебсторінкою.

DOM — це програмний інтерфейс браузера, який представляє HTML-документ
у вигляді дерева об'єктів. Коли браузер завантажує HTML-файл, він
не просто відображає текст на екрані — він будує в пам'яті структуру
даних, де кожен HTML-елемент є об'єктом з властивостями, методами
і посиланнями на сусідні елементи. JavaScript може читати й змінювати
будь-який вузол цього дерева.

Розглянемо простий HTML-документ:

    <html>
      <body>
        <h1 id="zaholovok">Привіт!</h1>
        <ul>
          <li>Перший пункт</li>
          <li>Другий пункт</li>
        </ul>
      </body>
    </html>

DOM-дерево для цього документа матиме таку структуру: корінь —
об'єкт document; його дочірній вузол — html; від html відгалужується
body; від body — два дочірні вузли: h1 і ul; від h1 — текстовий вузол
«Привіт!»; від ul — два вузли li, кожен із власним текстовим вмістом.

JavaScript отримує доступ до цього дерева через глобальний об'єкт
document. Метод document.querySelector() дозволяє знайти елемент
за CSS-селектором — точно так само, як ви б обрали елемент у таблиці
стилів. Метод document.getElementById() знаходить елемент за значенням
атрибута id. document.createElement() створює новий елемент. А метод
element.appendChild() додає створений елемент як дочірній вузол
до існуючого.

Важливо розуміти принципову різницю між HTML і DOM. HTML — це текст,
розмітка, файл на диску. DOM — це жива структура даних у пам'яті
браузера. Коли JavaScript змінює DOM — наприклад, змінює textContent
елемента або додає новий вузол — HTML-файл на диску не змінюється.
Змінюється лише те, що знаходиться в пам'яті браузера і відображається
на екрані. Це принципово важливо для розуміння того, як вебзастосунки
працюють і чому зміни, зроблені через JavaScript, не є постійними після
перезавантаження сторінки — якщо тільки вони не збережені у базі даних
або localStorage.


NODE.JS — JAVASCRIPT НА СЕРВЕРІ

2009 рік. Раян Дал виходить на сцену конференції JSConf EU і демонструє
проєкт, над яким працював кілька місяців. Проєкт називається Node.js.
За задумом, це середовище виконання JavaScript поза браузером —
побудоване на рушії V8 від Google, доповнене бібліотеками для роботи
з файловою системою, мережею і операційною системою.

Реакція спільноти була неоднозначною. Частина розробників відразу
побачила потенціал. Інша частина скептично знизала плечима: навіщо
використовувати JavaScript — мову, яку тоді вважали «іграшковою» —
для серйозного серверного коду? Відповідь прийшла впродовж кількох
наступних років. Node.js виявився надзвичайно ефективним для певного
класу задач — зокрема для систем із великою кількістю одночасних
підключень і операцій введення-виведення (I/O). Компанії на кшталт
Netflix, LinkedIn та Uber почали переводити частини своєї інфраструктури
на Node.js і звітували про суттєве підвищення продуктивності.

Що конкретно вміє Node.js? По-перше, створення HTTP-серверів і REST
API — і це найпоширеніший сценарій використання. Розглянемо мінімальний,
але повністю функціональний HTTP-сервер:

    const http = require('http');

    const server = http.createServer((zapyt, vidpovid) => {
      vidpovid.writeHead(200, { 'Content-Type': 'text/plain' });
      vidpovid.end('Привіт з Node.js!');
    });

    server.listen(3000, () => {
      console.log('Сервер запущено на порту 3000');
    });

Цей код слухає на порту 3000 і відповідає текстом на кожен запит.
Сім рядків — повноцінний вебсервер.

По-друге, Node.js має повний доступ до файлової системи через
вбудований модуль fs. Читання і запис файлів, створення директорій,
перейменування, видалення — всі ці операції доступні синхронно та
асинхронно. По-третє, Node.js підтримує підключення до баз даних —
як реляційних (PostgreSQL, MySQL), так і документоорієнтованих
(MongoDB). По-четверте, він є ідеальним інструментом для real-time
застосунків — чатів, систем сповіщень, онлайн-ігор — через вбудовану
підтримку WebSocket-протоколу.

Окремо варто сказати про npm — Node Package Manager. Це менеджер
пакетів, який постачається разом із Node.js і надає доступ до реєстру
пакетів npm. Станом на 2024 рік цей реєстр налічує понад 2,5 мільйона
пакетів — це найбільший реєстр відкритого програмного забезпечення
у світі. Будь-яка стандартна задача — валідація даних, робота з датами,
парсинг CSV, відправлення email, генерація PDF — має готове рішення
у вигляді npm-пакета.


JAVASCRIPT-ЕКОСИСТЕМА: ФРЕЙМВОРКИ ТА БІБЛІОТЕКИ

Самого лише JavaScript — мови та її стандартної бібліотеки — достатньо
для написання будь-якого вебзастосунку. Але у реальній розробці
практично завжди використовують фреймворки та бібліотеки: готові
інструменти, що вирішують типові задачі швидше, надійніше і з кращою
структурою, ніж це можна зробити з нуля.

React, створений компанією Meta у 2013 році, є бібліотекою для
побудови користувацьких інтерфейсів. Головна ідея React — компонентний
підхід: інтерфейс розбивається на самодостатні, повторно використовувані
компоненти, кожен із яких має власний стан і рендерить HTML залежно від
цього стану. React не є повноцінним фреймворком — він займається
лише шаром представлення (view) і не нав'язує рішень щодо маршрутизації
чи управління станом. Але саме ця мінімалізм у поєднанні з потужністю
зробила React найпопулярнішою JavaScript-бібліотекою: за даними Stack
Overflow 2024, нею користується більшість фронтенд-розробників.
React також є основою для React Native — фреймворку для мобільної
розробки, — а наш курс будується саме навколо React як центрального
інструменту.

Next.js — фреймворк, побудований поверх React і розроблений компанією
Vercel у 2016 році. Якщо React — це лише UI-шар, то Next.js — це
повноцінна платформа для розробки вебзастосунків: маршрутизація
на основі файлової системи, серверний рендеринг (SSR), статична
генерація сторінок (SSG), вбудована підтримка API-маршрутів,
оптимізація зображень і навіть власний механізм кешування. Саме
тому Next.js часто називають «повностековим фреймворком на JavaScript»:
з ним можна побудувати і фронтенд, і бекенд в межах одного проєкту.

Vue.js — фреймворк, створений у 2014 році Еваном Ю, колишнім
розробником Google. Позиціонується як «прогресивний» — тобто його
можна додавати у проєкт поступово, навіть до вже існуючого HTML,
а не переписувати все з нуля. Vue поєднує ідеї Angular та React,
але має значно нижчий поріг входу. У деяких країнах та індустріях —
зокрема в Китаї — Vue є домінуючим фреймворком.

Angular, розроблений Google і випущений у 2016 році (перепрацьована
версія оригінального AngularJS 2010 року), є повноцінним корпоративним
фреймворком. Він нав'язує чітку архітектуру, є «TypeScript-першим» —
тобто розроблявся з TypeScript з самого початку — і містить вбудовані
рішення для маршрутизації, форм, HTTP-клієнта і тестування. Angular
часто обирають великі компанії для масштабних проєктів, де важлива
стандартизація.

Express — мінімалістичний вебфреймворк для Node.js, що з'явився у 2010
році під управлінням організації OpenJS Foundation. Він є де-факто
стандартом для побудови REST API на Node.js: невеликий розмір,
простий API, величезна екосистема middleware-модулів. Більшість
навчальних матеріалів із серверного JavaScript починаються саме
з Express.

TypeScript, розроблений Microsoft і випущений у 2012 році, є
строго кажучи не фреймворком, а надбудовою над JavaScript —
але його місце в цій таблиці закономірне, бо він є невід'ємною
частиною сучасної JS-екосистеми. Про TypeScript — окремий підрозділ
нижче.

Electron, створений GitHub у 2013 році, вирішує задачу написання
настільних програм за допомогою JavaScript, HTML і CSS. Саме про
нього детальніше — у підрозділі про настільні застосунки.

Цей курс проходить шлях: JavaScript як фундамент, React як основний
UI-інструмент, Next.js як повностековий фреймворк, TypeScript як
інструмент для надійнішого коду і Node.js як серверна платформа.


JAVASCRIPT У МОБІЛЬНИХ ДОДАТКАХ

Мобільна розробка традиційно вважалася окремим, спеціалізованим
напрямком: Swift або Objective-C для iOS, Kotlin або Java для Android.
Дві різні платформи, дві різні мови, два різних кодових бази для
одного й того самого продукту. JavaScript змінив цю картину.

React Native, розроблений Meta і відкритий у 2015 році, є найбільш
поширеним рішенням для кросплатформної мобільної розробки на JavaScript.
Принципова відмінність React Native від звичайних «гібридних»
підходів полягає в тому, що він не обгортає вебсторінку в мобільний
контейнер: JavaScript-код компілюється у виклики нативних компонентів
платформи. Кнопка React Native — це справжня нативна кнопка iOS
або Android, а не відображення HTML-кнопки у WebView. Саме тому
застосунки на React Native виглядають і поводяться як нативні.

Серед найвідоміших застосунків, побудованих на React Native, — Facebook,
Instagram, Shopify і Discord. Це не демонстраційні проєкти: це
продукти з десятками мільйонів активних користувачів, що підтверджують
промислову придатність технології.

Expo — це платформа і набір інструментів, побудованих поверх React
Native. Якщо React Native сам по собі вимагає налаштування нативного
середовища збірки — Xcode для iOS, Android Studio для Android, — то
Expo значно спрощує цей процес. Вона надає готові API для камери,
геолокації, push-сповіщень, сенсорів та інших можливостей пристрою,
не вимагаючи написання жодного рядка нативного коду. Для більшості
стартапів і навчальних проєктів Expo є оптимальною точкою входу.

Ionic — дещо інший підхід. Він обгортає вебтехнології (HTML, CSS,
JavaScript) у нативний контейнер WebView і додає JavaScript-інтерфейси
для доступу до можливостей пристрою. Результуючий застосунок
технічно є вебзастосунком, що запускається як мобільний. Це дає
максимальну сумісність: Ionic підтримує Angular, React і Vue як
UI-фреймворки. Серед відомих компаній, що використовують Ionic, —
Airbus і MarketWatch.

Важливий практичний висновок: за оцінками ринку, близько 70% мобільних
розробників на React Native приходять із React-фону. Вивчивши React
для вебу, ви отримуєте більшу частину знань, необхідних для мобільної
розробки. Один набір навичок — два середовища.


JAVASCRIPT У НАСТІЛЬНИХ ЗАСТОСУНКАХ

Electron, розроблений GitHub і вперше публічно представлений у 2013 році,
є технологією, яка дозволяє створювати настільні програми для Windows,
macOS і Linux за допомогою JavaScript, HTML і CSS. Технічно Electron
поєднує Chromium — браузерний рушій, той самий, що використовується
у Chrome — з Node.js. Chromium відповідає за відображення інтерфейсу,
Node.js — за доступ до системних ресурсів.

Перелік застосунків, побудованих на Electron, вражає своєю якістю
і масштабом. Visual Studio Code — редактор коду від Microsoft, яким
користуються мільйони розробників щодня, — написаний на Electron.
Slack, Discord, Figma, 1Password, Notion — усі вони є Electron-застосунками.
Коли ви запускаєте ці програми, ви запускаєте JavaScript.

Electron має очевидні переваги: одна кодова база — три операційних
системи, використання вже відомих вебтехнологій, доступ до величезної
npm-екосистеми. Але має й відомі недоліки: порівняно великий розмір
дистрибутива (кожен Electron-застосунок постачається з повним рушієм
Chromium) і дещо вищий порівняно з нативними застосунками показник
споживання пам'яті.

Саме ці недоліки породили кілька альтернатив. Tauri — найпомітніша
з них — використовує Rust для бекенду замість Node.js і системний
WebView для відображення інтерфейсу замість Chromium, що дає значно
менший розмір дистрибутива і нижче споживання ресурсів. Фронтенд
залишається JavaScript-фреймворком за вибором розробника. NW.js —
ще один підхід, схожий на Electron, але з іншою моделлю інтеграції
Node.js. Neutralino є ультралегким варіантом: він використовує
вбудований у операційну систему WebView замість Chromium і пишається
розміром дистрибутива у кілька мегабайт замість сотень.


JAVASCRIPT СКРІЗЬ: IoT, ШТУЧНИЙ ІНТЕЛЕКТ ТА ХМАРНІ ОБЧИСЛЕННЯ

Сфера застосування JavaScript не зупиняється на браузері, сервері,
мобільних і настільних застосунках. Мова проникла глибше — у пристрої
Інтернету речей, у машинне навчання і у хмарні обчислення на кромці
мережі.

У сфері штучного інтелекту та машинного навчання JavaScript отримав
кілька серйозних інструментів. TensorFlow.js — це JavaScript-порт
відомої бібліотеки машинного навчання від Google, яка дозволяє
тренувати і запускати моделі машинного навчання безпосередньо у
браузері або в Node.js. Це не іграшковий інструмент: TensorFlow.js
підтримує навантаження реального рівня і може використовувати GPU
через WebGL для прискорення обчислень. Brain.js спрощує роботу з
нейронними мережами, надаючи зручний API для задач класифікації
та розпізнавання послідовностей. ml5.js побудований поверх TensorFlow.js
і орієнтований на художників, дизайнерів і початківців: з ним можна
розпізнавати пози тіла, обличчя або об'єкти на кількох рядках коду.
ONNX Runtime Web дозволяє запускати в браузері моделі у форматі ONNX —
відкритому стандарті для обміну між різними ML-фреймворками.

У сфері Інтернету речей (IoT) JavaScript також знайшов своє місце.
Johnny-Five — бібліотека для Node.js, яка дозволяє керувати
мікроконтролерами Arduino через JavaScript-код: читати показники
датчиків, керувати двигунами, мигати діодами. Espruino іде ще далі:
це мікроконтролер, на якому безпосередньо виконується JavaScript
без жодного проміжного шару. Raspberry Pi у поєднанні з Node.js —
класичний вибір для IoT-проєктів рівня хобі та прототипів: Python
там популярніший, але Node.js не поступається у можливостях. Браузерні
API WebBluetooth і WebUSB дозволяють вебсторінкам взаємодіяти
з фізичними пристроями безпосередньо через браузер.

У сфері хмарних обчислень і serverless-архітектури JavaScript є
де-факто стандартом. AWS Lambda — сервіс хмарних функцій від Amazon —
підтримує Node.js як одне з основних середовищ виконання. Cloudflare
Workers — один із найцікавіших нових продуктів у цьому просторі:
він виконує JavaScript на «кромці» мережі (edge computing), тобто
максимально близько до кінцевого користувача, у сотнях точок присутності
по всьому світу. Vercel і Netlify Functions роблять деплой серверних
функцій на JavaScript простим до межі: один файл коду — і функція
живе у хмарі. Google Cloud Functions підтримують JavaScript через
Node.js runtime.

Ця картина дозволяє зробити висновок, який важко спростувати:
JavaScript — єдина мова, що працює у браузері, на сервері, в хмарі,
на мобільному пристрої, у настільній програмі і на мікросхемі.
Жодна інша мова не охоплює такого широкого спектра середовищ із
однаковим синтаксисом і однаковими базовими концепціями.


TYPESCRIPT: JAVASCRIPT З ТИПАМИ

Розуміти сучасний JavaScript неможливо, не знаючи TypeScript. Не тому
що TypeScript замінив JavaScript — він його не замінив і не замінить.
А тому що у великих проєктах TypeScript став практично стандартом,
і розробник, який не розуміє його базових принципів, не зможе повноцінно
працювати з більшістю сучасних кодових баз.

TypeScript був розроблений компанією Microsoft і вперше публічно
представлений у 2012 році. Його головний архітектор — Андерс Гейлсберг,
який раніше створив мови Pascal і C#. TypeScript вирішує одну конкретну
проблему JavaScript: динамічна типізація зручна для малих проєктів,
але у великих кодових базах стає джерелом важко відтворюваних помилок.

Ключова властивість TypeScript, яку варто зрозуміти одразу: TypeScript —
це надбудова над JavaScript, а не окрема мова. Будь-який валідний
JavaScript-код є водночас валідним TypeScript-кодом. TypeScript лише
додає до JavaScript систему типів — і компілятор, який перевіряє
ці типи до запуску програми.

Розглянемо наочне порівняння. Ось функція додавання на JavaScript:

    function dodaty(a, b) {
      return a + b;
    }

    dodaty(2, 3);      // → 5
    dodaty('2', 3);    // → '23' — помилка знайдена лише в рантаймі

JavaScript не заперечує виклику dodaty('2', 3): він просто виконає
конкатенацію рядка та числа замість арифметичного додавання. Ніякого
повідомлення про помилку — тільки неправильний результат. Виявити
таку помилку у великій програмі може бути надзвичайно складно.

Ось та сама функція на TypeScript:

    function dodaty(a: number, b: number): number {
      return a + b;
    }

    dodaty(2, 3);      // → 5
    // dodaty('2', 3) → ПОМИЛКА при компіляції: Argument of type
    //                  'string' is not assignable to parameter
    //                  of type 'number'

Оголошення a: number і b: number — це анотації типів. Вони кажуть
TypeScript-компілятору: ці параметри повинні бути числами. Якщо
при виклику функції передати рядок — компілятор видасть помилку до
запуску програми. Код навіть не буде скомпільований у JavaScript.

TypeScript компілюється у звичайний JavaScript — після компіляції
залишається стандартний .js-файл без жодних TypeScript-анотацій.
Браузер і Node.js виконують цей звичайний JavaScript. TypeScript
повністю зникає після компіляції і існує лише на етапі розробки.

TypeScript сьогодні використовують Microsoft, Google, Airbnb, Slack,
Asana і команди розробників VS Code, Angular та Next.js. Він посідає
п'яту сходинку у рейтингу Stack Overflow 2024 і є одним із
найшвидше зростаючих інструментів у JavaScript-екосистемі.

У цьому курсі TypeScript вивчатиметься у Модулі 6 — після того, як
буде закладено міцний фундамент знань JavaScript та React.


JAVASCRIPT У ЦИФРАХ: ФАКТИ І СТАТИСТИКА

Числа іноді говорять переконливіше за будь-які пояснення. Тому варто
зупинитися і поглянути на масштаб JavaScript у 2024 році.

JavaScript є мовою програмування номер один за популярністю відповідно
до щорічного дослідження Stack Overflow Developer Survey — і зберігає
цю позицію дванадцять років поспіль. Не один рік, не три — дванадцять.
Жодна інша мова не має навіть близько такої тривалої безперервної
першості.

98% усіх вебсайтів у світі використовують JavaScript. Не мають намір
використовувати, не планують — вже використовують. Це означає, що
практично будь-яка сторінка, яку ви відкриваєте у браузері, виконує
JavaScript-код.

npm — реєстр пакетів для Node.js і JavaScript у цілому — налічує
понад 2,5 мільйона пакетів. Для порівняння: PyPI, реєстр пакетів
для Python, налічує близько 500 тисяч. Maven Central для Java —
близько 600 тисяч. npm перевершує всіх конкурентів у рази. Майже
для будь-якого завдання — від генерації UUID до реалізації OAuth —
вже є готовий, протестований пакет.

У всьому світі налічується понад 30 мільйонів JavaScript-розробників.
Для порівняння: Python — близько 15 мільйонів, Java — близько 8
мільйонів. Це найчисленніша програмістська спільнота на планеті,
що безпосередньо означає найбільшу кількість готових рішень,
навчальних матеріалів, відповідей на Stack Overflow і
open-source-проєктів.

Що стосується розподілу застосування за напрямками: вебфронтенд
займає 65,7% від усіх сценаріїв використання JavaScript, бекенд —
48,4%, мобайл — 19,8%, Desktop і IoT разом — 15,2%. Ці цифри
яскраво ілюструють: JavaScript вже давно перестав бути «лише
браузерною» мовою.


ЧОМУ ВАРТО ВИВЧАТИ JAVASCRIPT

Питання «чому» варто поставити прямо й відповісти на нього так само
прямо. Зрештою, якщо ви вже читаєте цей курс, рішення, можливо,
вже прийнято — але чітке розуміння причин завжди допомагає у моменти,
коли навчання дається важко.

JavaScript — найпопулярніша мова вже дванадцять років. Це не
примха ринку: за цим стоїть унікальна комбінація доступності, широти
застосування і зрілої екосистеми. Найбільша спільнота означає
найбільше ресурсів: навчальних матеріалів, готових рішень, відповідей
на питання. Будь-яка проблема, з якою ви зіткнетеся, майже гарантовано
вже обговорювалася і вирішувалася у Stack Overflow, GitHub або
YouTube.

JavaScript запускається буквально скрізь, де є сучасний браузер, —
а браузер є на кожному комп'ютері, телефоні й планшеті. Це означає,
що для першого кроку не потрібно нічого встановлювати. Відкрий
браузер, натисни F12, обери вкладку Console і пиши код. Результат
побачиш негайно.

Один набір навичок покриває весь стек. Знаючи JavaScript, ви можете
писати фронтенд (що відображається у браузері), бекенд (сервер
на Node.js), мобільний застосунок (React Native), настільну програму
(Electron) і хмарні функції (AWS Lambda). Жодна інша мова не надає
такої широти при єдиному інвестуванні часу в навчання.

Фінансовий аспект є важливим і реальним. За ринковими даними 2024
року, Junior JavaScript-розробник в Україні отримує від $800 до
$1500 на місяць, Middle — від $2000 до $4000, Senior — від $4000
до $8000 і вище. Це конкурентні рівні для українського ринку
незалежно від міста. А вихід на міжнародний фриланс або роботу
в іноземних компаніях відкриває ще вищі орієнтири.

Попит на ринку — стабільний і передбачуваний. JavaScript стабільно
входить у топ-3 найзатребуваніших технологій на DOU, LinkedIn,
Djinni та міжнародних платформах на кшталт Indeed і Glassdoor.
Веброзробка нікуди не зникне — вона лише ускладнюється і росте.

Нарешті, є аргумент, який важко виміряти, але легко відчути:
JavaScript відкриває можливість бачити результати своєї роботи
негайно. Написав функцію — викликав у консолі, побачив результат.
Змінив стиль кнопки — сторінка оновилася в реальному часі. Цей
безпосередній зворотний зв'язок — одна з причин, чому JavaScript
є такою приємною мовою для навчання.


ТВІЙ ПЕРШИЙ JAVASCRIPT-КОД

Теорія — важлива. Але JavaScript — практична мова, і найкраще її
відчути власноруч, тут і зараз, без жодних встановлень та налаштувань.

Відкрий браузер Chrome. Натисни клавішу F12 (або Fn+F12 на деяких
ноутбуках). Відкриється панель DevTools. Обери вкладку Console.
Ти бачиш рядок введення зі значком «>» або «>>». Це і є консоль
JavaScript: середовище, де можна виконувати код негайно.

Спробуй математику. Введи 2 + 2 і натисни Enter. Консоль відповість
4. Введи Math.PI — отримаєш число π з п'ятнадцятьма знаками після
коми. Введи [1, 2, 3].reverse() — отримаєш масив у зворотньому
порядку. JavaScript вміє виконувати математику, працювати з масивами
і видавати результат у мілісекунди.

Спробуй рядки. Введи 'Привіт' + ' Світ!' і натисни Enter. Отримаєш
'Привіт Світ!' — конкатенацію двох рядків. Введи typeof 42 і дізнаєшся,
що JavaScript вважає 42 значенням типу 'number'. Введи typeof 'текст' —
отримаєш 'string'.

А тепер напишімо першу власну функцію. Введи наступний код —
можна по рядку, натискаючи Shift+Enter після кожного рядка, щоб
не виконувати передчасно:

    function pryvitaty(imya) {
      return 'Привіт, ' + imya + '! Ласкаво просимо у JavaScript.';
    }

Натисни Enter. Тепер виклич функцію: pryvitaty('Оксана'). Отримаєш:
«Привіт, Оксана! Ласкаво просимо у JavaScript.» Це твоя перша
JavaScript-функція. Вона отримала аргумент, сформувала рядок
і повернула результат. Три рядки коду — і вже є щось справжнє.

Це перший дотик. Далі — значно глибше.


JAVASCRIPT ≠ JAVA: НАЙПОШИРЕНІША ПЛУТАНИНА

Перш ніж завершити цей розділ, необхідно раз і назавжди прояснити
одну з найпоширеніших плутанин у програмуванні — і, мабуть, одну
з найвартісніших маркетингових помилок в історії технологій.

JavaScript і Java — це дві абсолютно різні мови програмування. Спільне
між ними — лише маркетингове рішення 1995 року: назву «JavaScript»
обрали саме для того, щоб скористатися популярністю Java. Жодного
технічного зв'язку між ними немає. Спільноті вже давно приписують
фразу: «Java та JavaScript пов'язані так само, як автомобіль та
килимок». Це перебільшення, але воно передає суть: схожа назва,
повністю різна природа.

Розглянемо ключові відмінності.

Типізація: Java — статично типізована мова. Тип кожної змінної
оголошується явно і перевіряється компілятором до запуску. Написати
int x = "текст"; у Java — це помилка компіляції. JavaScript — динамічно
типізована мова: тип змінної визначається під час виконання і може
змінюватися.

Виконання: Java компілюється у байткод JVM (Java Virtual Machine) —
проміжний формат, не прив'язаний до конкретної апаратної архітектури.
Цей байткод виконується JVM — окремою програмою, яку необхідно
встановити. JavaScript інтерпретується або JIT-компілюється у браузері
чи Node.js без жодного додаткового встановлення для кінцевого
користувача.

Парадигма: Java є суворо об'єктно-орієнтованою мовою. Весь код
у Java існує всередині класів. Немає функцій поза класами, немає
процедурного коду верхнього рівня. JavaScript є мультипарадигмовою
мовою, де ООП, функціональне і процедурне програмування рівноправно
співіснують.

Синтаксис: обидві мови мають спільне коріння у C-подібному синтаксисі —
фігурні дужки, крапки з комою, конструкції if/else, for. Це єдине
реальне синтаксичне перетинання. Все інше різне.

Сфера застосування: Java домінує у корпоративному (enterprise) бекенді,
банківських системах і розробці для Android. JavaScript домінує у
вебі — і у фронтенді, і у бекенді — та у мобайлі через React Native.

Знати цю різницю — це не академічне знання. Розробники, які плутають
дві мови, виглядають непрофесійно. Скажіть «Java» замість «JavaScript»
на технічному інтерв'ю або в розмові з колегою — і ваш рівень буде
оцінений відповідно.


ПІДСУМОК РОЗДІЛУ

JavaScript — мова, яка народилася за десять днів і витримала тридцять
років. Сьогодні вона займає першу позицію за популярністю серед
розробників, запускається у 98% вебсайтів, має найбільшу спільноту
та найбільший реєстр пакетів у світі. Вона є інтерпретованою,
мультипарадигмовою, динамічно типізованою й однопотоковою за природою —
але завдяки Event Loop успішно справляється з асинхронністю.

У браузері JavaScript маніпулює DOM, обробляє події, надсилає мережеві
запити і відповідає за все, що рухається й оновлюється на сторінці.
На сервері — через Node.js — він будує REST API, читає файли і керує
базами даних. У мобільній розробці — через React Native — він генерує
нативні компоненти iOS та Android з єдиного JavaScript-коду. У
настільних програмах — через Electron — він є основою для VS Code,
Slack і Discord. У хмарі — через Cloudflare Workers і AWS Lambda —
він виконується на тисячах серверів у режимі serverless.

TypeScript — надбудова зі статичною типізацією — робить JavaScript
ще надійнішим у великих проєктах і є невід'ємною частиною сучасної
екосистеми. JavaScript і Java — різні мови, пов'язані лише назвою.

Наступний крок — навчитися встановлювати середовище розробки і
написати перший реальний код поза браузерною консоллю. Але фундамент
закладено: ви знаєте, що таке JavaScript, звідки він узявся і де живе.


ПРАКТИЧНІ ЗАВДАННЯ ДЛЯ САМОСТІЙНОЇ РОБОТИ

Матеріал цього розділу охоплює і теорію, і практику, і контекст —
а тому найефективніше засвоюється через власні дії та рефлексію.

Перше завдання — базовий рівень. Відкрийте консоль DevTools у будь-якому
браузері (F12, вкладка Console) і виконайте кілька виразів: обчисліть
результат математичної операції на ваш вибір, отримайте тип значення
typeof для числа, рядка і масиву, викличте метод 'рядок'.toUpperCase()
і перевірте результат. Зафіксуйте спостереження: яким є тип масиву
за результатом typeof? Це неочікувана поведінка — і вона має
пояснення, яке з'явиться у наступних розділах.

Друге завдання — середній рівень. Напишіть у тій самій консолі власну
функцію, яка приймає два числа і повертає їхню суму. Потім — функцію,
яка приймає рядок і повертає кількість символів у ньому. Зверніть увагу
на властивість .length. Викличте обидві функції з різними аргументами
і переконайтеся, що результати правильні.

Третє завдання — підвищений рівень. Відкрийте будь-який вебсайт.
У консолі введіть: document.querySelectorAll('a').length — це покаже
кількість посилань на сторінці. Потім введіть document.title —
отримаєте заголовок вкладки. Спробуйте змінити заголовок:
document.title = 'Мій заголовок' — і перевірте вкладку браузера.
Це і є маніпуляція з DOM у реальному часі: ви щойно змінили живу
вебсторінку через JavaScript.
